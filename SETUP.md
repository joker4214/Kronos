## Setup Steps

### 1. **Environment Setup**

```bash
cd /home/user/Kronos
npm install
```

### 2. **Database Setup (PostgreSQL)**

Create a PostgreSQL database:

```bash
createdb kronos
```

Update `.env` with your database URL:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/kronos"
```

Initialize Prisma:

```bash
npx prisma migrate dev --name init
```

### 3. **Google Calendar Integration**

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project
3. Enable Google Calendar API
4. Create OAuth2 credentials (Desktop application)
5. Download credentials JSON
6. Add to `.env`:

```env
GOOGLE_CALENDAR_ID="your-calendar-id@gmail.com"
GOOGLE_CLIENT_ID="your-id.apps.googleusercontent.com"
GOOGLE_CLIENT_SECRET="your-secret"
GOOGLE_REDIRECT_URI="http://localhost:3000/api/auth/google/callback"
```

### 4. **Claude API Key**

Get your API key from [Anthropic Console](https://console.anthropic.com)

```env
ANTHROPIC_API_KEY="sk-ant-..."
```

### 5. **Platform Integration** (Optional for MVP)

For manual posting (no auto-publish), just get your platform IDs:

```env
INSTAGRAM_BUSINESS_ACCOUNT_ID="your-id"
INSTAGRAM_ACCESS_TOKEN="your-token"
TIKTOK_BUSINESS_ACCOUNT_ID="your-id"
TIKTOK_ACCESS_TOKEN="your-token"
```

### 6. **Start Development Server**

```bash
npm run dev
```

Visit `http://localhost:3000`

## Workflow

### **Friday 10 AM: Content Generation**

1. Ideas from Google Calendar are synced
2. Claude generates platform-specific drafts
3. Drafts appear in dashboard for manual posting

### **Friday 8 AM: Metrics Collection**

1. System checks for posted content URLs
2. Pulls conversion data from tracking links
3. Shows what content performed best

## Database Schema

### CalendarIdea
- `id`, `title`, `description`
- `googleEventId` (linked to calendar event)
- `status` (pending, drafted, posted, archived)
- `scheduledDate`, `createdAt`, `updatedAt`

### ContentDraft
- `id`, `ideaId`, `platform` (instagram, tiktok, etc.)
- `content` (generated text)
- `status` (draft, posted, archived)
- `postUrl` (link to posted content)

### Metric
- `id`, `ideaId`, `platform`
- `conversions` (primary metric)
- `clicks`, `timeOnPage`, `engagement`
- `collectedAt` (when metric was collected)

### ScheduledTask
- Tracks Friday 8 AM and 10 AM job runs

## Scheduled Tasks

To enable scheduled jobs (Friday 8 AM & 10 AM), you can:

**Option 1: Deploy to Vercel with Cron Functions**
- Vercel automatically handles cron jobs
- Configure in `vercel.json`

**Option 2: Use External Service**
- Use AWS EventBridge, Google Cloud Scheduler, or EasyCron
- Call `/api/jobs/generate-content` and `/api/jobs/collect-metrics`

**Option 3: Local Development**
- Use `node-cron` in a separate process
- Create `scripts/scheduler.js`

## API Endpoints

### Ideas
- `GET /api/ideas` - List all ideas
- `POST /api/ideas` - Create idea
- `POST /api/calendar/sync` - Sync Google Calendar

### Content
- `POST /api/content/generate` - Generate drafts for an idea

### Drafts
- `GET /api/drafts` - List all drafts
- `PATCH /api/drafts/[id]` - Update draft (mark as posted)

### Metrics
- `GET /api/metrics` - List all metrics
- `POST /api/metrics` - Record new metric

### System
- `GET /api/system/status` - Check system health

## Platform-Specific Content

Claude automatically tailors content for each platform:

**Instagram**: Visual storytelling, emojis, hashtags (2200 char max)
**TikTok**: Catchy hooks, trending format (150 char max)
**LinkedIn**: Professional insights, thought leadership (3000 char max)
**Twitter**: Witty, concise, conversational (280 char max)

## Conversion Tracking

To track conversions:

1. Add UTM parameters to links: `?utm_source=instagram&utm_medium=social&utm_campaign=[idea_title]`
2. Use Google Analytics or your platform's conversion tracking
3. Manually enter conversion numbers in dashboard or via API
4. System collects metrics every Friday 8 AM

## Troubleshooting

**Database connection failed**
- Check PostgreSQL is running
- Verify DATABASE_URL in `.env`
- Run `npx prisma db push`

**Claude API errors**
- Verify ANTHROPIC_API_KEY in `.env`
- Check API key is valid and has sufficient balance

**Google Calendar sync not working**
- Verify Google Cloud credentials
- Check OAuth2 setup is complete
- Ensure calendar API is enabled
